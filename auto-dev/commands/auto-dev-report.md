# /auto-dev-report — 验收报告

生成标准化验收报告。

---

## 执行逻辑

### 1. 读取数据源

- 最近一次 `commands.test` 的输出
- `scripts/verify.sh` 的输出（如有）
- `.progress/status.md`
- `.auto-dev.yaml` 中的项目信息

### 2. 校验冻结文件完整性

对 `frozen_files` 中的每个文件计算 SHA-256，与 `frozen_checksums` 比对，在报告中标注校验结果。

### 3. 输出验收报告

```
╔══════════════════════════════════════════╗
║           auto-dev 验收报告              ║
╠══════════════════════════════════════════╣
║ 项目: <project.name>                    ║
║ 语言: <project.language>                ║
║ 日期: <当前日期>                         ║
╠══════════════════════════════════════════╣
║ 单元测试                                 ║
║   总用例: N    通过: X    失败: Y        ║
║   自愈轮次: Z                            ║
╠══════════════════════════════════════════╣
║ 集成测试                                 ║
║   状态: <通过/失败/未执行>                ║
║   自愈轮次: Z                            ║
╠══════════════════════════════════════════╣
║ 冻结文件校验: <通过/异常>                 ║
╠══════════════════════════════════════════╣
║ 结论: <全部通过 / 部分通过 / 未通过>      ║
╚══════════════════════════════════════════╝

## 详细结果

### 通过的测试
- [TC-001] <描述> — PASS
- [TC-002] <描述> — PASS

### 失败的测试（如有）
- [TC-003] <描述> — FAIL
  原因: <失败原因分析>

### 自愈记录
- 轮次 1: <修改内容和原因>
- 轮次 2: <修改内容和原因>

### 冻结文件校验
- test-specs/xxx.yaml — SHA-256 匹配 ✓
- scripts/verify.sh — SHA-256 匹配 ✓

### 遗留问题（如有）
- <问题描述>
```
